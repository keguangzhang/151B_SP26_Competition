# SFT data spec — target contract for `data/sft_corpus.jsonl`

The contract the final SFT corpus must satisfy. The data-prep notebook and the
mixer validate against this table; anything that fails an axis does not ship.

Targets are derived from [`docs/analysis/baseline-public-8k.md`](../analysis/baseline-public-8k.md) (the 8k-token run on
`public.jsonl`, 1,126 examples). Where a target deviates from the public
distribution, the deviation is justified inline. Numbers are starting points to
tune, not commitments — but a change to a target is a change to *this file*, not
an ad-hoc per-source choice.

Companion docs: [`docs/sft/pipeline.md`](pipeline.md) (the run plan), [`docs/sft/data-issues.md`](data-issues.md)
(defects in the current prepared sources). Index: [`docs/README.md`](../README.md).

---

## 1. Purpose

SFT should lift unified accuracy on `public.jsonl` / `private.jsonl` by
correcting the baseline's measured failure modes, **without** foreignizing the
model's native trace style. It is not a generic "train on more math" exercise.

The three failure modes the corpus must target:

1. **MCQ** — 50.4%. Whether this is format-repair or reasoning-repair is
   **unresolved** (see §6, open measurement M2).
2. **Multi-blank free-form** — 46.7% vs 63.0% single-blank.
3. **Weak topics** — number theory (30%), sequences/recurrences (35%),
   geometry (36%).

## 2. Provenance / teacher policy

**Self-distillation is primary.** Traces are generated by the baseline model
(`Qwen/Qwen3-4B-Thinking-2507`) itself, via rejection sampling: sample k times
per problem, keep traces that reach the correct answer, repair only the final
line. Rationale: the SFT target distribution is then the model's *own* output
distribution — no style shift, no capability gap, no trace-collapse risk.

External datasets (MATH, NuminaMath, OpenR1, AGIEval) are demoted from **trace
sources** to **problem sources**: they supply problem statements and verifiable
gold answers. Their original solutions are *not* training targets.

An external teacher (e.g. OpenR1 / R1 distill) is permitted **only** as a
targeted supplement for the hard tail — problems the baseline solves 0/k times,
where self-distillation yields no positive sample. Such rows are tagged
`provenance: "external_teacher"` so their share can be capped and audited
separately. License clearance is a prerequisite (open question in
`[pipeline.md](pipeline.md)`).

## 3. Output contract (every training target)

| Property | Requirement |
|----------|-------------|
| Trace schema | Reasoning inside `<think>...</think>`; final `\boxed{...}` **after** the closing tag ([D005](../log/decisions.md#d005--sft-assistant-schema-explicit-redacted_thinking-wrapper); artifact `data/qwen_thinking_trace.txt`) |
| Reasoning | Genuine multi-step CoT. No templated/synthetic filler (cf. [`data-issues.md`](data-issues.md) #1) |
| Final line — MCQ | `\boxed{<letter>}`, `<letter> ∈ [A-J]` |
| Final line — single-blank | single `\boxed{<value>}` |
| Final line — multi-blank | `\boxed{a}, \boxed{b}, ...` in question order |
| Prompt format | Exact `build_prompt(...)` + `apply_chat_template(...)` path from `dev.ipynb` / `submission.ipynb` |
| Language | English only — question and trace (cf. [`data-issues.md`](data-issues.md) #3, #5) |

### Trace length band

From the baseline: **correct** free-form traces average **6,204 chars**,
**incorrect** average **14,094**. Long traces correlate with failure, not
success — the model is stuck/reconsidering, not reasoning better.

| Task | Reject below | Target center | Reject above |
|------|-------------:|--------------:|-------------:|
| Free-form | 2,000 chars | ~5,000–8,000 chars | 12,000 chars |
| MCQ | 2,000 chars | TODO — refine from M2 sample | 12,000 chars |

This **overrides** the naive "longer trace is always better" filter in
`[pipeline.md](pipeline.md)` risk #1. The upper bound matters as much as the lower.

## 4. Input distribution targets

The corpus *prompt* distribution should track `public.jsonl`, with deliberate
upweighting on the weak buckets.

### 4.1 Task type

| | Public | SFT target | Rationale |
|--|------:|-----------:|-----------|
| MCQ | 33.3% (375) | 35–40% | Mild upweight — MCQ is the weaker bucket and the primary lever |
| Free-form | 66.7% (751) | 60–65% | — |

### 4.2 MCQ option count

| | Public | SFT target |
|--|------:|-----------:|
| 10-option | 89.6% (336/375) | ≥ 85% |
| Other (≤9-option) | 10.4% | ≤ 15% |

Distractors for synthetic option expansion must be type-coherent — sampled from
same-type problems, not the whole subset (cf. [`data-issues.md`](data-issues.md) #2).

### 4.3 Blank count (within free-form)

| | Public | SFT target | Rationale |
|--|------:|-----------:|-----------|
| Single-blank | 43.5% (327) | ~35% | — |
| Multi-blank | 56.5% (424) | ~65% | Upweight — weakest sub-bucket (46.7%) |

### 4.4 Topic

Track the public typed distribution, with weak topics multiplied **3×** then
renormalized.

| Topic | Public | Baseline acc | Weight |
|-------|-------:|-------------:|--------|
| Number theory | 2.0% | 30.4% | 3× (weak) |
| Sequences / recurrences | 6.7% | 34.7% | 3× (weak) |
| Geometry | 10.2% | 35.6% | 3× (weak) |
| Polynomials / algebra | 13.0% | 59.6% | 1× |
| Probability / stats | 7.3% | 50.0% | 1× |
| Integration | 4.9% | 74.5% | 1× (strong — keep, don't grow) |
| Limits / derivatives / linear algebra | ~4.3% | mixed | 1× |
| Other (catch-all) | 51.6% | 56.1% | 1× — **see caveat** |

**Caveat:** 51.6% of `public.jsonl` is the untyped "Other" bucket. The corpus
topic check cannot be fully validated until "Other" is subdivided (open
measurement M1). Until then, weak-topic upweighting is verifiable; the rest of
the topic match is approximate.

## 5. Quality filters & validation

Before a row enters `data/sft_corpus.jsonl`:

- Final-line regex passes for its task type.
- Trace length within the §3 band for its task type.
- English-only (no CJK in question or trace).
- Genuine reasoning — reject templated/degenerate traces.
- `template_tokens ≤ 7900` after chat-template rendering.
- No `public.jsonl` / `private.jsonl` question overlap (decontamination).
- For self-distilled rows: the trace's boxed answer matches the gold answer.

The mixer writes `data/sft_corpus_manifest.json` with per-axis realized
distributions so each §4 target can be checked against actuals.
